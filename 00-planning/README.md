# Planning

## Goal
Structure the MD-102 learning process with a hands-on, lab-driven approach.

## Scope
- Microsoft Entra ID
- Device enrollment
- Compliance policies
- Configuration profiles
- Application deployment
- Update management
- Security and endpoint protection

## Learning Approach
- Hands-on labs only (no passive learning)
- Every lab is documented
- All errors are tracked in issues.md
- Each setup must be reproducible
- Small, focused tasks per day

## Initial Objectives
- Tenant setup (if available)
- VM setup
- Device enrollment

## Structure
- 00-planning → planning and decisions
- 01-* → topic-based lab folders
- assets → screenshots and outputs

## Notes
Tenant-based labs may be postponed depending on environment constraints.
